# How to Request Changes

This guide explains how teachers can request changes to the school activities website without needing to know how to code.

## Creating a New Issue

When you need to make a change to the website, you create an "issue" on GitHub. Think of it as a request form that tells the system what you need.

### Step 1: Go to the Issues Page

1. Go to the repository on GitHub
2. Click on the "Issues" tab at the top
3. Click the green "New Issue" button

### Step 2: Choose the Right Template

You'll see several templates to choose from. Pick the one that matches what you need:

**Example:** Adding "Photography Club" or "Volleyball Team"

**What you'll need to provide:**
- Activity name
- Description of what students will do
- Category (Sports, Arts, Academic, Community, or Technology)
- Schedule (which days and times)
- Maximum number of students

**Example:** Changing Chess Club's meeting time or updating Soccer Team's description

**What you'll need to provide:**
- Which activity needs updating
- What needs to change
- The new information
- Why the change is needed

#### üé® UI Enhancement Request
Use this when you want to change how the website looks.

**Example:** "Make the buttons bigger" or "Use our school colors"

**What you'll need to provide:**
- What part of the website to change
- How it looks now
- How you want it to look
- Why the change is needed

#### üêõ Bug Report
Use this when something on the website isn't working correctly.

**Example:** "When I click Register Student, nothing happens" or "The search box doesn't find activities"

**What you'll need to provide:**
- What went wrong
- Steps to reproduce the problem
- What you expected to happen instead
- How severe the problem is

#### ‚ú® Feature Request
Use this when you want to add completely new functionality to the website.

**Example:** "Add the ability to export student lists to Excel" or "Add a calendar view"

**What you'll need to provide:**
- Description of the new feature
- What problem it solves
- Who will use it
- How they would use it

### Step 3: Fill Out the Form

After selecting a template, you'll see a form with various fields. Fill in all the required fields (marked with a red asterisk *).

**Tips:**
- Be as specific as possible
- If you're not sure what to put in a field, write what you know
- You can always add more information in the comments later

### Step 4: Submit the Issue

1. Review your information
2. Click the green "Submit new issue" button
3. Your request is now in the system!

## What Happens Next?

Once you submit an issue:
1. GitHub Copilot (an AI coding assistant) or a developer will review your request
2. They'll make the necessary changes to the website
3. You'll get notifications when the work is complete
4. You can add comments to the issue if you need to provide more information

## Getting Help

If you're not sure which template to use or need help filling it out:
1. Choose the template that seems closest to what you need
2. Fill in what you can
3. Add a note in the "Additional Context" section explaining that you need help
4. Someone will reach out to clarify

## Examples

### Example 1: Adding a New Activity
**Template:** Add New Activity
- **Activity Name:** Photography Club
- **Description:** Students will learn basic photography skills, camera operation, and photo editing
- **Category:** Arts
- **Days:** Tuesday, Thursday
- **Time:** 3:30 PM - 5:00 PM (15:30 - 17:00)
- **Max Students:** 15

### Example 2: Fixing a Bug
**Template:** Bug Report
- **Problem:** When I try to register a student for Chess Club, I get an error message
- **Steps:** 1) Log in as a teacher, 2) Click "Register Student" on Chess Club, 3) Enter student email, 4) Click Register
- **Expected:** Student should be added to the activity
- **Severity:** High - I can't register any students

### Example 3: Changing the Look
**Template:** UI Enhancement Request
- **What:** Make the header use our school colors
- **Where:** Header/title area
- **Current:** The header is blue
- **Desired:** The header should be white with lime green text
- **Why:** To match our school branding
